## ğŸ“ FAQ

**Q: What if my feature doesn't get used much?**
A: Impact isn't just usage. Even a niche feature shows you can ship production code. Plus, you control the narrative in interviews.

**Q: Can I work with a partner on features?**
A: Some features can be pair-programmed, but each student must own their own feature for their portfolio.

**Q: What if I want to build something not on the backlog?**
A: Propose it! If it fits the skills for that phase and helps VWC, instructors will likely approve.

**Q: What happens to features after I graduate?**
A: They stay in production! VWC maintains them, and future students may enhance them. Your name stays in git history.

**Q: Can I showcase these features to employers?**
A: Absolutely! That's the point. You can demo them, show the code, explain the impact. VWC encourages it.

**Q: What if my feature breaks in production?**
A: Part of the learning! We'll help you debug and fix it. Real engineers deal with production bugs.
